Identify Housing Maintenance Code Potential Violators in New York City#

The New York Attorney General (NYAG) is actively combating various harmful landlord practices such as tenant harassment, deed theft, and bank fraud, which adversely impact tenants and homeowners. By leveraging open data publicly available through NYC Open Data, this notebook uses data analysis tools to identify, describe, and visualize characteristics of landlords engaging in harmful behaviors.

Data#

In this section, we import from the data sources publicly through NYC Open Data [1]. The NYC Open Dataâ€™s catalog contains information for each dataset, including data dictionaries, which will leverage to inform our analysis. For this assessment, we devote our focus to the data sources below:

  • All Housing, Preservation, and Development (HPD) data (available: http://on.nyc.gov/1YlIN2t)

    • Registrations (also available via Socrata: http://bit.ly/1Uvviry)

    • Registration Contacts (also available via Socrata: http://bit.ly/2rjoGJg)

    • Violations (also available via Socrata: http://bit.ly/1Ue5dju)

    • Pluto Building (available: http://on.nyc.gov/1RRCnGO)

On data access, the datasets can be retrieved via multiple methods, particularly through the Socrata Open Data API. However, we opted for latest static downloadable CSV files retrieved from the catalog to the ../data relative path.

According to the documentation, Violations are issued in four classes: Class A (non-hazardous), Class B (hazardous), Class C (immediately hazardous) and Class I (information orders). To identify violators, we will look violations for B and C.

Hotstops of Open Violations#

In this step, we identify hotspots of open violations for Class B (hazardous), Class C (immediately hazardous). This criteria should indicate a population most severely impacted by the absolute number of open violations.

Show code cell source Hide code cell source 
data = (
    VIOLATIONS[
        (VIOLATIONS["ViolationStatus"] == "Open")
        & (VIOLATIONS["hex_id"] != "0")
        & (VIOLATIONS["Class"].isin(["B", "C"]))
    ]
    .groupby(["hex_id"])["ViolationID"]

Merging VIOLATIONS and PLUTO#

In this step, we will enrich VIOLATIONS with additional features from PLUTO, which describe the lot in richer detail. The BBL will be used for left merging. Next, we test out a straightforward hypothesis: determining the buildings (via BuildingID and the BBL (Borough Block Lot)) with the highest number of hazardous violations (of class B and C) to identify potential violators.

Show code cell source Hide code cell source 
VIOLATIONS_BY_BBL = (
    VIOLATIONS.query('Class == "B" or Class == "C"')
    .groupby(["BBL", "RegistrationID", "BuildingID"])["ViolationID"]
    .nunique()
    .to_frame("violation_count")
    .reset_index()
    .sort_values(["violation_count"])
)

VIOLATIONS_BY_BBL = pd.merge(
    VIOLATIONS_BY_BBL,
    PLUTO,
    left_on=["BBL"],
    right_on=["bbl"],
    how="inner",
).sort_values("violation_count", ascending=False)

Findings#

By leveraging open data publicly available through NYC Open Data [1] and by cross referencing data Housing Maintenance Code Violations and Registration Contacts, this analysis demonstrated an approach to identify individuals associated with the highest number of hazardous violations.

In total, we identified 21 individuals associated with 10 buildings and hazardous violations. The criteria to select these individuals is straightforward: tabulate and rank the number of hazardous violations of buildings associated with each individual. A top 10 can be selected from this group, which may be used to guide further investigations by the NYAG.

In addition to the violations dataset, we explored combining additional features from NYC Pluto. However, the combined dataset did not seem to enrich the analysis after a preliminary inspection. On the other hand, time permitting, this combined dataset could be explored to enhance the selection criteria. For instance, to consider not only the number of violations, but other features such as building area and residential area in order to maximize the potential impact of an investigation.

Additionally, time permitting, the analysis could go beyond the exploratory phrase. After data modelling and assessing assumptions and limitations of the input data, the dataset could be prepared for a machine learning approach. For instance, based on characteristics of the block, lot, building, violation history and landlord(s), a model could predict how likely a violation is to be reported and shortlist and flag cases that may need further investigation as the violations dataset is updated daily.
